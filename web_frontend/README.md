# LegalDocInspector Web Frontend

Веб-интерфейс для системы автоматизации обработки юридических документов и генерации исковых заявлений.

## Технологии

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Backend**: Flask (Python 3.11+)
- **Интеграция**: Основной бэкенд-сервер на порту 5001

## Установка

1. Убедитесь, что у вас установлен Python 3.11+

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Убедитесь, что основной бэкенд-сервер запущен на `http://localhost:5001`

## Запуск

Запустите Flask приложение:

```bash
python app.py
```

Или с использованием Flask CLI:

```bash
export FLASK_APP=app.py
flask run --port=5000
```

Приложение будет доступно по адресу: `http://localhost:5000`

## Использование

1. Откройте браузер и перейдите на `http://localhost:5000`

2. **Этап 1: Загрузка документов**
   - Выберите дату конца просрочки
   - Выберите тип компании
   - Загрузите документы для каждого набора:
     - Договор (PDF, DOC, DOCX)
     - Претензия (PDF, DOC, DOCX)
     - Excel справка о задолженности (XLS, XLSX, XLSM) - можно несколько файлов
   - При необходимости добавьте дополнительные наборы документов
   - Нажмите "Отправить на сервер"

3. **Этап 2: Заполнение данных**
   - Проверьте и заполните данные о суде
   - Заполните данные об истце (ИНН заполняется автоматически)
   - Проверьте данные об ответчике (заполняются автоматически)
   - Заполните данные о договорах
   - Нажмите "Произвести расчёты"

4. **Этап 3: Проверка данных иска**
   - Проверьте цену иска и госпошлину
   - Выберите вид услуги
   - Проверьте данные о претензиях
   - Нажмите "Подтвердить правильность данных"

5. **Этап 4: Скачивание документов**
   - Скачайте готовый иск
   - Скачайте расчёт к иску

## Особенности

- Сохранение состояния между перезагрузками страницы (LocalStorage)
- Автоматическое заполнение данных по ИНН
- Поддержка множественных наборов документов
- Адаптивный дизайн для мобильных устройств
- Обработка ошибок с понятными сообщениями

## API Endpoints

Flask приложение предоставляет следующие endpoints:

- `GET /` - главная страница
- `GET /api/parse-inn/<inn>` - получение данных по ИНН
- `POST /api/calculate-duty` - расчёт госпошлины
- `POST /api/proxy/parse` - проксирование запроса для парсинга документов
- `POST /api/proxy/calculate-penalty` - проксирование запроса для расчёта пени
- `POST /api/proxy/create-doc` - проксирование запроса для создания иска
- `POST /api/proxy/create-calculating-table` - проксирование запроса для создания расчёта
