# Отладка: Где посмотреть список претензий, отправляемых на бэкенд

## Проблема
Ошибка на бэкенде: "В списке претензий в данных нет ни одной претензии"

## Где посмотреть список претензий на фронтенде

### 1. В консоли браузера (Developer Tools)

Откройте консоль браузера (F12) и выполните:

```javascript
// Посмотреть текущее состояние претензий
console.log('Текущие претензии:', appState.formData.lawsuitInfo.claims);

// Посмотреть все данные о иске
console.log('Данные о иске:', appState.formData.lawsuitInfo);

// Посмотреть полные данные, которые будут отправлены
console.log('Данные для отправки:', {
    claims: appState.formData.lawsuitInfo.claims,
    lawsuitInfo: appState.formData.lawsuitInfo
});
```

### 2. В коде JavaScript

Список претензий формируется в следующих местах:

1. **Функция `renderClaims()`** (строка ~753 в `app.js`):
   - Извлекает претензии из `appState.formData.result.table_parser_result`
   - Сохраняет в `appState.formData.lawsuitInfo.claims`

2. **Функция `generateDocuments()`** (строка ~824 в `app.js`):
   - Собирает претензии из полей формы перед отправкой
   - Проверяет наличие претензий
   - Логирует данные в консоль перед отправкой

### 3. Структура данных

Претензии хранятся в:
```javascript
appState.formData.lawsuitInfo.claims = [
    "№ 123456 от 01.01.2024",
    "№ 123457 от 02.01.2024"
]
```

При отправке на бэкенд данные передаются в структуре:
```javascript
{
    claim_data: {
        lawsuit_info: {
            claims: ["№ 123456 от 01.01.2024", ...]
        }
    }
}
```

### 4. Проверка перед отправкой

В функции `generateDocuments()` добавлена проверка:
- Если список претензий пустой, показывается ошибка
- Данные логируются в консоль перед отправкой

### 5. Отладка в реальном времени

Добавьте в консоль браузера:
```javascript
// Отслеживание изменений претензий
Object.defineProperty(appState.formData.lawsuitInfo, 'claims', {
    set: function(value) {
        console.log('Претензии обновлены:', value);
        this._claims = value;
    },
    get: function() {
        return this._claims || [];
    }
});
```

## Возможные причины ошибки

1. **Претензии не были извлечены из документов** - проверьте `appState.formData.result.table_parser_result`
2. **Поля претензий не заполнены** - проверьте наличие элементов в `#claims-container`
3. **Претензии были удалены** - проверьте, что `appState.formData.lawsuitInfo.claims` не пустой массив

## Решение

Если претензии не найдены автоматически:
1. Поля для ввода претензий будут показаны с предупреждением
2. Заполните претензии вручную в полях формы
3. Претензии будут собраны из полей перед отправкой
