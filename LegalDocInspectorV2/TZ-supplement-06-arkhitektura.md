- Для аудита и логирования?

### Q-6.1.4. Хранилище файлов — детали

## 6.2. Диаграмма потоков

### Q-6.2.1. Управление состоянием пакета

Как BFF отслеживает состояние обработки пакета?
- In-memory (потеря при перезапуске)?
- В Redis/БД (надёжно, но сложнее)?
- В самой очереди задач (по статусу задач)?

Какие состояния проходит пакет?
```
created → files_uploaded → extracting → extracted → form_editing → calculating → calculated → generating → documents_ready
```
Или упрощённая модель?

### Q-6.2.2. Обработка частичных результатов

Когда BFF возвращает результат фронтенду?
- Дождаться завершения **всех** задач извлечения (включая самую долгую OCR)?
- Или отдавать результаты по мере готовности (streaming/progressive loading)?

Если progressively — как фронтенд обновляет форму при получении новых данных (WebSocket, SSE, polling)?
