## 8.2. Ответ после извлечения

### Q-8.2.1. Полная схема объекта `form`

ТЗ содержит только пример с `plaintiff`, `defendant`, `claims`. Нужна **полная и исчерпывающая** JSON Schema для объекта `form`:

**Истец (plaintiff):**
- `name` — полное наименование
- `shortName` — сокращённое наименование?
- `ogrn`
- `inn`
- `kpp`
- `legalAddress` — юридический адрес
- `actualAddress` — фактический адрес (если отличается)?
- `directorName` — ФИО руководителя?
- `directorPosition` — должность руководителя?
- `bankName`, `bik`, `checkingAccount`, `corrAccount` — банковские реквизиты?
- `phone`, `email` — контактные данные?
- Какие из этих полей обязательны, какие опциональны?

**Ответчик (defendant):**
- Аналогичный набор полей или сокращённый?
- Откуда берутся данные ответчика — из договора, из ЕГРЮЛ, или пользователь вводит вручную?

**Требования (claims):**
- Какова структура одного claim? Что он содержит?
- Привязан ли claim к конкретному documentSet?

### Q-8.2.2. Полная схема `extracted` для каждого типа документа

Для каждого типа документа нужна точная структура извлечённых данных:

**Из договора (contract):**
```json
{
  "contractNumber": "string",
  "contractDate": "date",
  "contractSubject": "string?",
  "contractAmount": "decimal?",
  "penaltyRate": "decimal?",
  "penaltyType": "enum?",
  "paymentTermDays": "integer?",
  "jurisdictionClause": "string?",
  ...?
}
```

**Из претензии (preClaim):**
```json
{
  "preClaimDate": "date",
  "preClaimNumber": "string?",
  "preClaimAmount": "decimal?",
  "deliveryDate": "date?",
  "deliveryMethod": "string?",
  ...?
}
```

**Из справок о задолженности (debtCertificates):**
```json
{
  "periods": [
    {
      "periodStart": "date",
      "periodEnd": "date",
      "debtAmount": "decimal",
      "description": "string?"
    }
  ],
  "totalDebt": "decimal",
  ...?
}
```

**Из ЕГРЮЛ (egrul):**
```json
{
  "fullName": "string",
  "shortName": "string?",
  "ogrn": "string",
  "inn": "string",
  "kpp": "string?",
  "legalAddress": "string",
  "directorName": "string?",
  "directorPosition": "string?",
  "registrationDate": "date?",
  ...?
}
```

Нужен полный перечень полей для каждого типа.

### Q-8.2.3. Статусы извлечения

Какие значения может принимать `status` в `documentSetResults`?
- `ok` — все поля извлечены успешно?
- `partial` — часть полей извлечена, часть нет?
- `failed` — ничего не извлечено?
- `pending` — ещё в обработке?
- Есть ли другие статусы?

### Q-8.2.4. Маппинг extracted → form

Как результаты извлечения из разных документов (ЕГРЮЛ, договор, претензия, справки) объединяются в единый объект `form`?
- Автоматически на стороне BFF?
- Есть ли приоритет источников (например, ИНН из ЕГРЮЛ приоритетнее чем из договора)?
- Что если одно поле извлечено из нескольких источников с разными значениями?

## 8.3. Результат расчёта

### Q-8.3.1. Полная схема расчёта

Пример в ТЗ содержит только `lines` и `totals`. Нужна полная схема:
- Какие типы `lines` возможны (основной долг, неустойка, проценты по 395 ГК, госпошлина, судебные расходы)?
- Для каждого line: нужны ли дополнительные поля помимо `label` и `amount` (период, ставка, формула, основание)?
- Нужна ли разбивка по documentSets (расчёт отдельно по каждому комплекту)?
- Какова структура `totals` — только `grandTotal` или подытоги по категориям?

### Q-8.3.2. Представление денежных сумм

- Строка (`"100000.00"`) или число (`100000.00`)?
- Всегда два знака после запятой?
- Валюта указывается явно (`"currency": "RUB"`) или всегда рубли?
- Нужно ли поле для суммы прописью?

### Q-8.3.3. Представление дат

- Формат ISO 8601 (`"2026-04-11"`) или иной?
- Таймзона — UTC или Moscow (Europe/Moscow)?
- Для `calculatedAt` — ISO 8601 с таймзоной (`"2026-04-11T12:00:00+03:00"`) или UTC (`"2026-04-11T09:00:00Z"`)?
